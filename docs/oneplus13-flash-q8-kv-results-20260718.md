# OnePlus 13 Flash Attention and Q8 KV Cache Result

## Scope

This is an initial same-device CPU comparison between the F16 KV-cache reference and `Flash Attention=On` with a Q8_0 KV cache. It uses the same Qwen3 4B Instruct Q4_K_M GGUF, fixed Chinese benchmark prompt, default system prompt, temperature `0.3`, maximum output `96`, CPU runtime profile, batch `512`, and ubatch `512`.

It is not an accelerator or calibrated energy claim. The F16 and Q8 stages were taken sequentially on a warm phone, so only rows with five valid measured samples on both sides are treated as controlled throughput evidence.

The Q8 stage has five valid 6-thread samples while the F16 stage has only three. This is useful sustained-thermal evidence, but it does not establish a 6-thread throughput winner because the sample counts and thermal histories differ.

## Memory and CPU evidence

Post-run `dumpsys meminfo` shows:

| Metric | F16 KV | Q8 KV | Change |
| --- | ---: | ---: | ---: |
| Total PSS (KB) | 6,184,951 | 5,667,065 | -517,886 (-8.37%) |
| Native Heap (KB) | 3,675,964 | 3,128,524 | -547,440 (-14.89%) |

The first 60 seconds of the F16 stage were captured with device Simpleperf: 113,994 CPU-clock samples, zero lost. Inclusive hotspots are OpenMP `__kmp_invoke_microtask` / `ggml_graph_compute_thread`, Q4_K repack matrix multiplication, and `ggml_gemv_q4_K_8x8_q8_K`. Model reload also appears in the profile because the formal benchmark deliberately uses a fresh model/context per slot and reports load time separately.

## Evidence and limitations

Raw stage JSON/CSV/HTML, device manifests, Simpleperf data/reports, `meminfo`, thermal, battery, gfxinfo, logcat, and UI-tree snapshots are preserved under ignored `benchmarks/output/` directories. The stage comparison HTML and CSV are generated by `tools/compare-benchmark-stages.ps1` using the model hash, device fingerprint, prompt fingerprints, token limit, and sampling temperature as its comparability key.

- Q8 KV is not yet a blanket default: it improved the fully valid 2/4-thread rows in this run, while the 6-thread row needs a cold, randomized five-versus-five repeat.
- Both 8-thread blocks are thermal-invalid artifacts, not zero-performance or zero-memory results.
- The result is CPU/KleidiAI/OpenMP only; no GPU, HTP, QNN, Vulkan, or calibrated power claim is made.
